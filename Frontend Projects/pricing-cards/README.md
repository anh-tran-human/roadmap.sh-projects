# Pricing Cards

## Goal

Create a row of pricing cards using only HTML and CSS. Pricing cards are a common UI pattern used on marketing and SaaS websites to compare plans side by side and guide users to the right tier.

## Requirements

- [x] Equal heights: All three cards should be the same height even when their feature lists are different lengths, so the buttons line up at the bottom.

- [x] Hover state: The button should have a clear hover and focus state. Do not rely on color alone - change the background or add a subtle shadow.

- [x] Box model: Use padding for inner spacing and gap on the row for spacing between cards. Avoid mixing margin and gap to space siblings.

- [x] Typography hierarchy: The price should be the most prominent element on the card; the plan name and feature list should sit clearly below it.
